---
title: 'Login'
description: 'Authenticate with email and password to receive a JWT.'
api: 'POST /api/auth/login'
---

Validates the provided credentials and returns a signed JWT. Use the token in the `Authorization` header for all endpoints that require authentication.

<Info>
  No authentication is required to call this endpoint.
</Info>

## Request body

<ParamField body="email" type="string" required>
  The email address associated with the account.
</ParamField>

<ParamField body="password" type="string" required>
  The account password in plain text. Compared against the stored bcrypt hash server-side.
</ParamField>

## Response

<ResponseField name="token" type="string" required>
  Signed JWT. Pass this value as a Bearer token in subsequent requests:
  `Authorization: Bearer <token>`
</ResponseField>

<ResponseField name="usuario" type="object" required>
  <Expandable title="properties" defaultOpen={true}>
    <ResponseField name="id" type="number">
      Internal user ID.
    </ResponseField>
    <ResponseField name="nombre_usuario" type="string">
      The account's username.
    </ResponseField>
    <ResponseField name="email" type="string">
      The account's email address.
    </ResponseField>
    <ResponseField name="nombre_completo" type="string">
      Display name, or `null` if not set.
    </ResponseField>
    <ResponseField name="rol" type="string">
      Account role. One of `invitado`, `registrado`, or `admin`.
    </ResponseField>
    <ResponseField name="nivel_cinefilo" type="string">
      Cinephile level. One of `explorador`, `entusiasta`, `experto`, or `null`.
    </ResponseField>
    <ResponseField name="onboarding_completo" type="boolean">
      Whether the user has completed onboarding.
    </ResponseField>
    <ResponseField name="activo" type="boolean">
      Whether the account is active.
    </ResponseField>
    <ResponseField name="creado_en" type="string">
      ISO 8601 timestamp of account creation.
    </ResponseField>
    <ResponseField name="actualizado_en" type="string">
      ISO 8601 timestamp of last update.
    </ResponseField>
  </Expandable>
</ResponseField>

<Tip>
  Store the returned token securely (e.g., in `localStorage` or an HTTP-only cookie) and attach it to every authenticated request as `Authorization: Bearer <token>`.
</Tip>

## Example

<RequestExample>

```bash cURL
curl --request POST \
  --url http://localhost:3000/api/auth/login \
  --header 'Content-Type: application/json' \
  --data '{
    "email": "jdoe@example.com",
    "password": "S3cur3P@ssw0rd"
  }'
```

</RequestExample>
